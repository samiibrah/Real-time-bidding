## Architecture

This is a DSP-side real-time bidding simulation. All bidding logic lives in `rtb_sim/`:

**`auction.py`** — The environment (opponent). `AuctionEnvironment` generates `Impression` objects with ground-truth `true_pcvr` (never revealed to the agent) and runs auctions using either first-price or second-price mechanics. Competitor bids are drawn from a log-normal distribution. Floor prices cause automatic losses if the agent bid is below them.

**`agent.py`** — The DSP agent and its sub-components:
- `PCVREstimator`: logistic regression with Platt scaling; uses cyclical encodings for hour/day-of-week, OHE for device/vertical, log-transform for user recency. Retrains online every 200 buffered impressions. Returns `0.02` cold-start prior before fitting.
- `BudgetPacer`: throttle-based spend control. Computes `pace_ratio = actual_spend / expected_spend` and maps it to a participation rate (0.3–1.0). The agent calls `pacer.record_impression(spent=0.0)` at bid time, then separately increments `pacer._spent` in `record_result`.
- `BidShader`: adapts shade factor from win price coefficient of variation. Active only in first-price mode.
- `BiddingAgent`: orchestrates all three. Core bid formula: `bid = pCVR × target_CPA × 1000` (CPM units). Returns `None` to opt out of an auction.

**`landscape.py`** — `BidLandscapeEstimator`: fits a log-normal competitor bid distribution using censored MLE. Wins contribute the uncensored likelihood `f(x|μ,σ)`; losses contribute only `P(X > c|μ,σ)` (right-censored survival). Refits every 25 new observations after reaching `min_obs`. Used in `simulation.py` to track landscape stats but not yet wired into bid decisions.

**`simulation.py`** — Orchestration. `run_simulation()` runs the full loop and returns a DataFrame with one row per entered auction. `compare_strategies()` benchmarks flat CPM, oracle pCVR, second-price pCVR, and first-price shaded strategies under identical conditions.

## Key Invariants

- All monetary values are in **CPM (cost per thousand impressions)**. The conversion to per-impression cost is `clearing_price / 1000`.
- `true_pcvr` on `Impression` is generated by `AuctionEnvironment._true_pcvr()` and is the ground truth the agent can never directly access — it must estimate it with `PCVREstimator`.
- `BiddingAgent.record_result()` contains a latent bug: the `bid` variable used in the loss branch (`self.shader.record_loss(bid)`) references an undefined local — the loss-recording logic for `BidShader` is currently effectively disabled. The `record_win` path for `BidShader` is also commented out.
- The landscape estimator is fitted and queried in `run_simulation()` but its output (`landscape_win_prob`) is only logged — it does not feed back into bid decisions.
